1
요구사항 정리하기
쿠폰 어드민에 필요한 기능과 화면을 Claude에게 설명합니다
완료 →
이렇게 말했어요
쏘카 쿠폰 어드민 페이지를 만들어줘. 정책 관리, 발행 요청, 사용 현황, 충돌 감지, 변경 이력 페이지가 필요해. 다크 테마로 만들어줘.
핵심 포인트
구체적으로 말할수록 좋아요. "쿠폰 관리 페이지 만들어줘"보다 페이지 구성, 필요한 데이터, 디자인 스타일을 함께 알려주면 훨씬 정확한 결과물이 나옵니다.
포함된 기능
정책 목록 테이블 발행 요청 폼 사용 현황 통계 충돌 감지 카드 변경 이력 타임라인 사이드바 네비게이션
2
디자인 시스템 잡기
색상, 폰트, 컴포넌트 스타일을 정의합니다
완료 →
이렇게 말했어요
다크 테마로 만들어줘. 배경은 거의 검정에 가깝게, 카드는 약간 밝게. 파란색을 메인 강조색으로, 상태별로 초록/노랑/빨강 쓰고. 폰트는 Noto Sans KR + JetBrains Mono.
적용된 디자인 토큰
5단계 배경 계층: #0a0c10 → #12151c → #181c25 → #1e2330 → #252a36
상태 색상: 파랑(정보) · 초록(성공) · 노랑(경고) · 빨강(에러) · 보라(정보)
폰트: Noto Sans KR (본문) + JetBrains Mono (숫자, 코드)
3
정책 관리 페이지 만들기
통계 카드, 알림 바, 필터, 테이블을 포함한 메인 페이지
완료 →
이렇게 말했어요
정책 관리 페이지에 상단에 통계 카드 4개 (활성 정책, 발행량, Cap 임박, 자동 중지), 경고 알림 바, 그리고 테이블에는 정책명, 유형, 상태, Cap 진행바, 적용 요일, 기간, 버전을 보여줘. 모쿠폰-자쿠폰 계층 구조도 표현해줘.
만들어진 컴포넌트
통계 카드 (상단 컬러 라인) 경고/위험 알림 바 검색 + 필터 셀렉트 Cap 프로그레스 바 요일 칩 부모-자식 행 구조 페이지네이션
4
발행 요청 폼 만들기
9개 섹션의 쿠폰 생성 폼 — 접기/펼치기, 차량 검색, 요일 선택
완료 →
이렇게 말했어요
쿠폰 발행 요청 폼을 만들어줘. 기본 정보, 발행·Cap 설정, 기간·요일·시간, 차종·차량, 존·지역, 운행 타입, 대상·타겟팅, 가격제·추가설정, 알림·연동 — 총 9개 섹션. 각 섹션은 접기/펼치기 가능하게. 차종은 태그 검색으로, 요일은 버튼으로 선택.
인터랙션 요소
섹션 접기/펼치기 할인 유형별 단위 변경 요일 토글 버튼 차종 검색 + 태그 추가 토글 스위치 Cap 자동중지 설정
5
충돌 감지 & 이력 페이지
정책 간 충돌 경고 카드와 변경 이력 타임라인
완료 →
이렇게 말했어요
충돌 감지 페이지 — 조건 중복(심각), 매출 잠식(주의), 레거시 간섭(주의) 3개 충돌을 카드로 보여줘. 심각도 배지 포함. 변경 이력은 Cap 초과, 수정, 생성, 종료 유형별 아이콘과 타임스탬프로.
핵심 포인트
충돌 카드는 심각도에 따라 테두리 색상이 달라져요 (빨강=심각, 노랑=주의). 이력 아이템은 유형별 아이콘 색상으로 직관적 구분 (초록=생성, 파랑=수정, 노랑=Cap, 빨강=중지).
